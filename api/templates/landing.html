Por que usar este projeto?

Arquitetura pensada para portfólio e produção: API serverless, banco com RLS e front leve.

Links dinâmicos

O slug permanece; você troca o destino quando precisar, sem reimprimir materiais.

Rastreio de leituras

Cada acesso ao caminho curto incrementa o contador no Supabase, visível no dashboard.

Auth + RLS

Cada usuário vê apenas os próprios QR Codes; políticas no PostgreSQL reforçam a segurança.
